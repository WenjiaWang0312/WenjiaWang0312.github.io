  1. 首屏定位更锋利                                                                                                                                                                                         
     把首页开头压成“你是谁 + 你做什么 + 当前状态”两三句，再配几个清晰入口，比如 Publications、                                                                                                              
  Projects、CV。                                                                                                                                                                                            
  2. 首页顺序更像研究主页                                                                                                                                                                                   
     建议从“简历式堆叠”改成更清楚的结构：                                                                                                                                                                   
     Hero -> Research Scope -> Featured Work -> News -> Services                                                                                                                                            
  3. Selected Papers 不要只是纯列表                                                                                                                                                                         
     更适合做成 3 到 4 个代表作卡片，放 teaser 图、1 句话贡献、venue，以及 Project / Paper / Code 链                                                                                                        
  接。                                                                                                                                                                                                      
  4. 强化研究主线                                                                                                                                                                                           
     你已经有 Real2Sim 和 Sim2Real，这是很好的骨架，但可以再写得更具体一点：                                                                                                                                
     每条线解决什么问题，代表工作是什么。                                                                                                                                                                   
  5. News 保持 milestone 风格                                                                                                                                                                               
     保留最重要的 4 到 6 条，不要变成流水账。最好让每条一眼看出是 paper、code、acceptance、project                                                                                                          
  page 之类的里程碑。                                                                                                                                                                                       
  6. 合作入口单独显式化                                                                                                                                                                                     
     你现在有欢迎学生和合作的表达，但埋在段落里了。可以单独做一个 Openings / Collaboration 小节，说明                                                                                                       
  你欢迎什么方向、怎么联系。                                                                                                                                                                                
  7. 减少纯文字列表感                                                                                                                                                                                       
     你的站点更适合“研究展示”，不是博客，所以应该多用卡片、缩略图、按钮、短句，少用很长的列表和嵌套条                                                                                                       
  目。                                                                                                                                                                                                      
  8. 首页内容尽量和 collections 同步                                                                                                                                                                        
     比如 Selected Papers、News、Projects 最好逐步从 /_publications、projects/ 这些内容来源里统一维                                                                                                         
  护，减少重复更新。
